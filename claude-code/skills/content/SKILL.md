## Voice and Tone

Write like you're texting a friend who's interested in tech. Casual, direct, no buzzwords. Never use phrases like "game-changer", "unlock your potential", or "in today's fast-paced world." Those are instant credibility killers.

Short sentences. Short paragraphs. One idea per paragraph max.

No em dashes. No "X - description" list formats. No corporate speak.

## First Run Adaptation

1. Check if there's a brand guide or content style doc in the project
2. Look for previous content in a content/ or posts/ directory to learn the existing voice
3. Note the target audience from any README or about page

## Platform Guidelines

Instagram captions: hook in the first line, keep it under 150 words, end with a question or CTA. No hashtag spam. 3-5 relevant hashtags max.

TikTok captions: super short. 1-2 sentences that complement the video. The video does the heavy lifting.

Twitter/X posts: under 280 characters when possible. Make it punchy. Hot takes do well but only if you actually believe them.

LinkedIn: slightly more professional but still conversational. Lead with a bold statement or surprising insight. 3-5 short paragraphs max.

Blog posts: can be longer and more detailed. Include specific steps, code snippets, or examples when relevant. Break into scannable sections.

## What NOT to Do

Never start with "Hey everyone!" or "So I've been thinking..."
Never use em dashes.
Never list things with "X - description" format.
Never end with "What do you think? Let me know in the comments!" unless it's genuinely a question you want answered.
Never use more than 2 emojis per post.

## Output

When creating content, always specify which platform it's for and provide 2-3 variations so the user can pick their favorite.

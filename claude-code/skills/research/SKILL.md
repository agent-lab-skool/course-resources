## Process

1. Search for the topic using multiple queries from different angles. Don't stop at the first result.
2. Look for primary sources. Documentation, official announcements, published papers. Avoid regurgitated blog posts that just summarize other articles.
3. Cross-reference claims across sources. If only one source says something, flag it as unverified.
4. Look for recent information. Prioritize sources from the last 6 months when the topic is fast-moving.
5. Search for dissenting opinions or known limitations. Good research includes the downsides.

## Output Format

Write findings to a markdown file at research/[topic-slug].md with this structure:

**Summary** (3-5 sentences of the key findings)

**Key Points** (the most important things learned, each with a brief explanation)

**Details** (deeper exploration of each key point with source references)

**Open Questions** (things that aren't clear or need more investigation)

**Sources** (URLs of everything referenced, with access dates)

## Rules

Be honest about uncertainty. If the research is inconclusive, say so.
Don't pad the output. If there's not much to find, a short report is fine.
Always include the date of the research since things change fast.
Distinguish between facts (documented, multiple sources) and opinions (single source, blog posts).
If a source contradicts another source, present both sides and note the conflict.
